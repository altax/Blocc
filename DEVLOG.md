### Новые модули бота

**`chat-hype-detector.ts`** — Детектор хайпа чата (NEW)
- Скользящее окно 30 сек, отслеживает velocity (msg/sec)
- Topic clustering: когда 30%+ чата пишет одно слово — это хайп
- Emoji storm detection: 50%+ сообщений с Twitch эмоутами
- Question storm: чат засыпает стримера вопросами
- Результат: `currentLevel` 0-10, `isHot` флаг, `dominantTopic`

**`game-state-machine.ts`** — CS2 Game State Machine (NEW)
- Извлекает структурированный game state из текстовых описаний vision/speech
- Отслеживает: `roundPhase`, `momentType`, `momentIntensity` (0-10), счёт CT/T, карту
- Обнаруживает: ace, clutch, bomb_planted, knife_kill, eco_win, pistol_round, AWP highlight
- Рассчитывает `sessionMood`: hyped/tense/tilted/chill/comeback/stomp на основе history побед/поражений
- Интенсивность затухает со временем (decay 0.3/сек)

**`session-memory.ts`** — Долгосрочная память сессии (NEW)
- Хранит дугу сессии: заметные моменты, настроение, цитаты стримера
- Tracks bot mood на основе последних 10 quality scores
- Сохраняет топ-10 лучших сообщений сессии
- Следит за темами которые уже упоминал (не повторяться)
- Классифицирует цитаты стримера: funny/tilt/hype/question/info

### Апгрейды существующих модулей

**`vision-analyzer.ts`** — Структурированный CS2 Vision Prompt
- Заменён слабый prompt "Describe in 1-2 sentences" на полный JSON-промпт
- Gemini теперь возвращает: round_phase, moment_type, intensity, ct_score, t_score, map, streamer_hp, bomb_planted, is_clutch, description
- Автоматически обновляет game-state-machine после каждого кадра
- Добавлен `quickIntensityCheck()` для быстрого 0-10 рейтинга без полного анализа

**`context-builder.ts`** — Умный контекст
- Добавлен CS2 game state раздел с emoji-индикаторами (🔴 БОМБА, 🔥 КЛАТЧ)
- Добавлен hype level с уровнем и velocity
- Добавлена session memory (настроение, длительность, что уже упоминал)
- `selectContextualPatterns()` — умный выбор паттернов по контексту:
  - Высокий хайп → короткие паттерны (вирусные)
  - Clutch/ace → реакшн-паттерны (монка, боже)
  - Death/loss → copium/sarcasm паттерны
  - Default → weighted by quality×log(freq)

**`response-generator.ts`** — Контекстно-умный генератор
- `buildSystemPrompt()` адаптирует промпт под момент:
  - ACE → "Напиши что-то с восхищением и хайпом"
  - КЛАТЧ → "Тревога + облегчение (монка... КЛАТЧ PogChamp)"
  - KNIFE → "Хайп, смех, эмоуты"
  - 5 поражений подряд → "Лёгкий copium, по-человечески"
- Инжектирует последний `prompt_delta` из рефлексий (самокоррекция)
- Умная инъекция паттернов: при хайпе вероятность растёт до 55%
- `shouldRespond()` — быстрый путь без LLM:
  - Intensity ≥ 8 → немедленно YES (экономит токены)
  - Hype ≥ 8 → немедленно YES
  - Bomb planted → 70% шанс YES
  - Только при обычных моментах → спрашивает LLM

**`orchestrator.ts`** — Hot Trigger System
- Добавлен второй таймер: `hotTriggerTimer` каждые **3 секунды** (vs 15с decision loop)
- При intensity ≥ 8 или hype ≥ 8 — немедленная реакция, задержка всего 0.5-3с (vs 3-10с обычная)
- `startBot()` инициализирует все новые модули (hype, game state, session)
- `stopBot()` сохраняет итоги сессии в лог
- `getBotStatusPayload()` теперь включает `game_state` и `hype_state`
- `onSpeechTranscript()` / `onVisionSummary()` — публичный API для routes

### Архитектурное решение: почему именно так

**Почему Hot Trigger вместо просто короче cooldown:**
Cooldown контролирует "не слишком часто". Hot Trigger — это отдельный канал для экстренных реакций. Бот может молчать 3 минуты (нормально), но мгновенно среагировать на эйс. Это именно человеческое поведение.

**Почему быстрый путь в shouldRespond без LLM:**
Каждый вызов GPT-4o-mini на shouldRespond стоит ~$0.00001 и занимает 300-800мс. При intensity ≥ 8 (ace/clutch) ответ очевиден — любой живой зритель напишет. LLM нужен только для неоднозначных ситуаций.

**Почему контекстный выбор паттернов:**
Раньше: рандомный паттерн из топ-40 с 30% шансом. Теперь: под clutch момент инжектируем "монкаС... КЛАТЧ PogChamp", а не "збс пуш красавы". Значимость паттерна зависит от контекста.

### Задача 1 — Умный CS2 Vision Prompt (структурированный game state)

**Проблема сейчас:**
`vision-analyzer.ts` шлёт Gemini обобщённый промпт: *"Describe what's happening in 1-2 sentences"*.
Gemini отвечает: *"A player is shooting at enemies."* — это бесполезно.

**Что сделать:**
- Новый промпт в `vision-analyzer.ts` возвращает структурированный JSON:
  ```json
  {
    "game": "cs2",
    "map": "de_mirage",
    "side": "ct",
    "score": "12-3",
    "round": 14,
    "player_hp": 1,
    "event_type": "clutch_1v3" | "ace" | "death" | "win_round" | "loss_round" | "bomb_planted" | "defuse" | "knife" | "idle",
    "tension_level": 95,
    "description": "Стример один против троих, HP=1, раунд на матчбол"
  }
  ```
- `event_type` и `tension_level` (0–100) — главные поля для принятия решений
- `context-builder.ts` форматирует этот JSON в читаемый контекст для GPT
- Функция `buildContextString()` использует `event_type` напрямую в заголовке контекста

**Файлы:** `artifacts/api-server/src/lib/bot-engine/vision-analyzer.ts`, `context-builder.ts`

### Задача 2 — Событийная реакция (event-driven loop вместо 15-секундного таймера)

**Проблема сейчас:**
`orchestrator.ts` запускает `runDecisionLoop()` каждые 15 секунд через `setInterval`.
ACE длится 4 секунды. Чат реагирует через 1-2 секунды. Бот ждёт до 15 сек → момент мёртв.

**Что сделать:**
- Новый класс `EventEmitter` / функция `onHighTensionEvent()` в `orchestrator.ts`
- При получении vision-ответа с `tension_level >= 80` или `event_type` ∈ `[ace, clutch_1v3, death, knife]` → немедленно триггерить `runDecisionLoop()` без ожидания таймера
- `shouldRespond()` при таких событиях возвращает `true` с вероятностью 90% (вместо обычных 25%)
- Задержка перед отправкой: 1-4 секунды (имитация человека, который только что увидел момент)
- Обычный 15-секундный цикл остаётся для фоновых реакций

**Файлы:** `artifacts/api-server/src/lib/bot-engine/orchestrator.ts`, `vision-analyzer.ts`

### Задача 3 — Эмоция речи + спайк чата

**Проблема сейчас:**
- Whisper транскрибирует *"НОООЖ НООЖЖЖ Я В ТЕБЯ ВЕРИЛ"* → попадает как строка в контекст. Система не знает что стример **кричит** = надо реагировать прямо сейчас.
- Спайки чата (10+ сообщений за 3 сек) игнорируются полностью — а это лучший сигнал что произошло что-то важное.

**Что сделать:**

**3a — Анализ эмоции речи:**
- После Whisper-транскрипции → GPT-4o-mini (или эвристика) оценивает `excitement_level` 0–100:
  - Ключевые слова: АААА, НЕТ НЕТ НЕТ, ОРУ, НОООЖ → 90+
  - Заглавные буквы + восклицания → 70+
  - Тихая речь → <30
- Если `excitement_level >= 70` → немедленный триггер (как в задаче 2)

**3b — Детектор спайков чата:**
- `context-builder.ts` отслеживает количество входящих IRC-сообщений за скользящее окно 5 секунд
- Если > 8 сообщений за 5 сек → `chatSpike: true` в контексте
- `orchestrator.ts` при `chatSpike` запускает `runDecisionLoop()` немедленно
- Дополнительно: анализировать о чём пишет чат (доминирующее слово/эмоция) и включать в промпт

**Файлы:** `artifacts/api-server/src/lib/bot-engine/context-builder.ts`, `orchestrator.ts`

## Цель проекта
Создать ИИ-бота для Twitch, который наблюдает за CS2-стримами (видео + аудио), читает чат и пишет сообщения, неотличимые от живого русского зрителя. Главная задача — стать популярным стримером, поэтому бот должен максимально точно имитировать поведение аудитории топовых русских CS2 стримеров.

---

## Принципы антидетекта
- Бот НЕ должен быть обнаружен как бот ни при каком условии
- Паттерны берутся исключительно из реального русского CS2 чата
- Задержки и частота сообщений имитируют реального зрителя (~раз в 1-3 мин)
- 15% шанс промолчать даже при активном триггере (как реальный человек)
- Симуляция опечаток у 18% сообщений (дублирование/пропуск букв)
- Инъекция реальных паттернов в 30% сообщений
- Cooldown рандомизирован, рекомендуется 60–120с

## Архитектурные решения
- **Паттерны в БД с приоритизацией** — score = (ru×3) + (cs2×2) + frequency. Русские CS2 паттерны всегда в топе.
- **Инъекция паттернов в промпт** — топ-20 реальных фраз инжектятся в системный промпт GPT-4o-mini при каждой генерации. Это ключевой механизм имитации.
- **Anti-detect через рандомизацию** — задержки, вероятностное молчание, редкие опечатки. Фиксированные паттерны сразу видны модераторам.
- **IRC anonymous для обучения** — при сборе паттернов используем justinfanXXXXX, никакого риска бана аккаунта.
- **Bulk learn последовательно** — каналы обучаются по очереди с паузой 2с между ними, чтобы не получить ban от Twitch за спам подключений.
